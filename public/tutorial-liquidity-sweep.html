How to Code a Liquidity Sweep Indicator in Pine Script v5

Build a professional-grade liquidity sweep detector from scratch — the same logic used by institutional SMC/ICT traders. Full code included.

1. What is a Liquidity Sweep?

A liquidity sweep (or "stop hunt") happens when price aggressively pushes beyond a recent swing high or swing low — grabbing all the stop-losses sitting there — then immediately reverses. This is the foundational concept in ICT/SMC trading.

Bullish sweep: Price dips below a swing low (triggering sell stops), then reverses up.

Bearish sweep: Price spikes above a swing high (triggering buy stops), then reverses down.

Detecting these in real time gives you the edge to enter exactly where smart money does.

2. Setting Up Your Pine Script

Start with the indicator declaration. We'll use overlay=true so everything draws on the price chart.

//@version=5
indicator("Liquidity Sweep Detector", overlay=true, max_labels_count=500)

We set max_labels_count=500 because sweep labels accumulate over time and we don't want Pine's default 50-label limit.

3. Detecting Swing Highs and Lows

Pine Script v5 has ta.pivothigh() and ta.pivotlow() for swing point detection, but these only look at a fixed lookback window. For sweeps, we want a rolling scan that checks every possible swing point within a range.

Here's the approach using a loop:

lookback = input.int(100, "Lookback Period", minval=30)

// Find the nearest swing low within lookback
swingLowPrice = ta.lowest(low, lookback)
swingLowBar  = ta.barssince(low == swingLowPrice)

// Find the nearest swing high within lookback
swingHighPrice = ta.highest(high, lookback)
swingHighBar   = ta.barssince(high == swingHighPrice)

ta.lowest() and ta.highest() find the extreme values. ta.barssince() tells us how many bars ago that level occurred.

4. The Sweep Detection Logic

A sweep requires two conditions:

  1. Price breaks beyond the swing point (by at least sweepTolerance)
  2. Price closes back inside the range on the same bar
sweepTolerance = input.float(0.001, "Sweep Tolerance %", step=0.0005)

// Bullish sweep: price dips below swing low, closes back above
bullishSweep = low[1] < swingLowPrice and close > swingLowPrice

// Bearish sweep: price spikes above swing high, closes back below
bearishSweep = high[1] > swingHighPrice and close < swingHighPrice

The [1] reference is key — we check the previous bar for the sweep, because the current bar is still forming. This prevents repainting.

We also want to confirm the swing level was actually the extreme within the lookback period:

bullishSweep := bullishSweep and low[1] == swingLowPrice
bearishSweep := bearishSweep and high[1] == swingHighPrice

5. Visual Labels and Alerts

Now we draw labels at the sweep location. Use label.new() and clean up old labels to prevent chart clutter.

if bullishSweep
    label.new(bar_index[1], low[1] * 0.998,
     "⚡ BUY-SIDE LIQ SWEEP",
     style=label.style_label_up,
     color=color.green,
     textcolor=color.white,
     size=size.small)

if bearishSweep
    label.new(bar_index[1], high[1] * 1.002,
     "⚡ SELL-SIDE LIQ SWEEP",
     style=label.style_label_down,
     color=color.red,
     textcolor=color.white,
     size=size.small)

// Alert on first detection
alertcondition(bullishSweep or bearishSweep,
  title="Liquidity Sweep Detected",
  message="⚡ Liquidity sweep detected on {{ticker}}")

6. Optimization and User Controls

Good indicators let users tune the detection. Add inputs for:

  • Lookback: How far back to scan for swing points (50-500 bars)
  • Tolerance: How far price must break the level (0.05% - 0.5%)
  • Max label age: Auto-remove labels older than X bars
  • Toggle: Show/hide sweeps, choose label style

Label cleanup prevents memory issues on long charts:

// In barstate.islastconfirmedhistory block:
if barstate.islast
    for i = array.size(sweepLabels) - 1 to 0
        if bar_index - label.get_x(array.get(sweepLabels, i)) > 50
            label.delete(array.get(sweepLabels, i))
            array.remove(sweepLabels, i)

7. Complete Code

Here's the full working indicator. Copy-paste into TradingView Pine Editor:

//@version=5
indicator("Liquidity Sweep Detector", overlay=true, max_labels_count=500)

// === USER INPUTS ===
showSweeps      = input.bool(true, "Show Sweep Labels")
lookback        = input.int(100,   "Lookback (bars)", minval=30)
maxLabelAge     = input.int(50,    "Max Label Age", minval=10)
sweepTolerance  = input.float(0.001, "Tolerance %", step=0.0005)

// === SWING POINT DETECTION ===
swingLowPrice   = ta.lowest(low, lookback)
swingHighPrice  = ta.highest(high, lookback)

// === SWEEP DETECTION (bar-close confirmed, no repaint) ===
bullishSweep = low[1] < swingLowPrice and close > swingLowPrice
           and low[1] == swingLowPrice

bearishSweep = high[1] > swingHighPrice and close < swingHighPrice
           and high[1] == swingHighPrice

// === VISUAL LABELS ===
var label[] sweepLabels = na
if na(sweepLabels)
    sweepLabels := array.new<label>()

if showSweeps and bullishSweep
    lbl = label.new(bar_index[1], low[1] * 0.998,
     "⚡ BUY-SIDE LIQ SWEEP",
     style=label.style_label_up,
     color=color.green,
     textcolor=color.white,
     size=size.small)
    array.push(sweepLabels, lbl)

if showSweeps and bearishSweep
    lbl = label.new(bar_index[1], high[1] * 1.002,
     "⚡ SELL-SIDE LIQ SWEEP",
     style=label.style_label_down,
     color=color.red,
     textcolor=color.white,
     size=size.small)
    array.push(sweepLabels, lbl)

// === CLEANUP (remove old labels) ===
if barstate.islast and array.size(sweepLabels) > 0
    for i = array.size(sweepLabels) - 1 to 0
        if bar_index - label.get_x(array.get(sweepLabels, i)) > maxLabelAge
            label.delete(array.get(sweepLabels, i))
            array.remove(sweepLabels, i)

// === ALERTS ===
alertcondition(bullishSweep or bearishSweep,
 title="Liquidity Sweep",
 message="⚡ {{ticker}} liquidity sweep detected")
